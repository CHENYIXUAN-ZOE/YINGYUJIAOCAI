# 口语对练内容结构与页面架构

## 1. 当前项目定位

当前更合理的产品定位不是“PDF 教材解析工具”，而是一个围绕教材单元内容组织的 `口语对练 Web 工具`。

用户核心路径应为：

1. 选择教材版本、年级、册次
2. 进入某个单元或复习单元
3. 查看本单元对练主题、重点句型、重点词汇
4. 开始一轮师生口语对练
5. 在结束后查看本轮练习覆盖情况和待改进项

这意味着内容参考的作用是：

- 作为教材内容来源
- 作为 prompt 设计依据
- 作为页面展示素材
- 不是逐字段照搬的数据终态

## 3. 建议采用的统一内容模型

建议把所有教材内容统一成四层结构：

1. 教材
2. 单元
3. 对练配置
4. 教学素材

### 3.1 教材层 `book`

建议字段：

- `book_id`
- `publisher`
- `edition`
- `grade`
- `term`
- `volume_name`
- `start_point`
- `display_name`
- `status`

示例：

```json
{
  "book_id": "pep_grade2_fall",
  "publisher": "人教版",
  "edition": "一年级起点",
  "grade": 2,
  "term": "上册",
  "volume_name": "二年级上册",
  "start_point": "一年级起点",
  "display_name": "人教版二年级上册",
  "status": "draft"
}
```

### 3.2 单元层 `unit`

建议字段：

- `unit_id`
- `book_id`
- `unit_code`
- `unit_name`
- `unit_type`
- `topic`
- `scene`
- `task_description`
- `teaching_goal`
- `sort_order`

说明：

- `unit_type` 至少区分 `regular` 和 `review`
- `topic` 用于页面展示和 prompt 生成
- `scene` 用于归类，例如 `family`、`classmates`、`park`

### 3.3 对练配置层 `conversation_config`

建议字段：

- `role_name`
- `student_profile`
- `language_policy`
- `topic_redirect_rule`
- `response_limit`
- `correction_policy`
- `clarification_policy`
- `difficulty_adaptation_rule`
- `opening_line`
- `turn_rule`
- `must_use_patterns`
- `must_cover_vocab`
- `scene_scope`
- `conversation_steps`
- `warnings`

说明：

- 这一层是 prompt 的核心配置层
- `conversation_steps` 不要只存一整段文本，建议拆成数组
- `warnings` 用于存英语输出限制、句数限制、禁止暴露 AI 身份等规则

### 3.4 教学素材层 `teaching_assets`

建议至少拆成 4 类：

- `vocabulary_items`
- `sentence_patterns`
- `dialogue_samples`
- `review_rules`

#### `vocabulary_item`

建议字段：

- `word`
- `part_of_speech`
- `meaning_zh`
- `phonetic_uk`
- `phonetic_us`
- `example_sentences`
- `tags`

#### `sentence_pattern`

建议字段：

- `pattern`
- `usage_note`
- `examples`
- `required`

#### `dialogue_sample`

建议字段：

- `title`
- `scenario`
- `content_en`
- `content_zh`
- `speakers`

#### `review_rule`

复习单元建议单独建模，不直接等同普通单元：

- `source_units`
- `review_topics`
- `task_sequence`
- `randomization_rule`
- `coverage_goal`

- `单元任务` -> `task_description`

- `对话样例` -> `dialogue_samples`

`北师大版-学习样例+Prompt要素.xlsx` 可以直接映射为：

- `GPT角色` -> `role_name`

- `重点词汇` -> `must_cover_vocab`
- `预置首句` -> `opening_line`
- `单元任务` -> `task_description`

## 5. 页面信息架构

建议先做 5 个页面，够支撑第一版外部可访问产品。

### 5.1 首页

目标：

- 让用户知道这是一个教材驱动的英语口语对练工具
- 快速进入教材选择

核心模块：

- 产品标题和价值说明
- 教材版本入口
- 年级和册次选择
- 推荐单元入口

### 5.2 教材页

目标：

- 让用户看到当前教材下有哪些单元

核心模块：

- 教材信息
- 单元列表
- 常规单元和复习单元区分
- 每个单元的主题和一句话任务摘要

### 5.3 单元详情页

目标：

- 让用户在开始对练前先理解“这一课练什么”

核心模块：

- 单元标题
- 对练任务
- 重点句型
- 重点词汇
- 对话样例节选
- 开始对练按钮

说明：

- 这一页很重要，它能把参考内容转成用户可理解的学习目标
- 不建议用户直接从首页跳进聊天页

### 5.4 对练进行页

目标：

- 承载主要交互

核心模块：

- 当前单元信息
- 聊天区
- 重点句型提示区
- 重点词汇提示区
- 当前进度提示
- 结束本轮按钮

交互原则：

- AI 每次尽量只推进一个问题
- 对低龄学生始终保持短句
- 修正错误时不打断流程

### 5.5 练习结果页

目标：

- 告诉用户这轮练到了什么、还缺什么

核心模块：

- 对练主题回顾
- 已覆盖重点句型
- 已覆盖重点词汇
- 表达问题提醒
- 建议再练内容
- 再来一轮按钮

说明：

- 这一页会决定产品是否有“学习闭环”
- 即使第一版不做复杂评分，也至少要做覆盖情况回显

## 6. 后台或内容管理侧的信息架构

如果后面要做可维护内容库，建议内部至少有 3 类管理页：

### 6.1 教材管理

- 新增教材
- 编辑教材元信息
- 上下架

### 6.2 单元管理

- 编辑单元任务
- 编辑句型
- 编辑词汇
- 编辑样例对话
- 配置普通单元或复习单元

### 6.3 Prompt 配置管理

- 角色设定
- 学生画像
- 轮次规则
- 句数限制
- 首句配置
- 风险约束

## 7. 第一版产品建议只做哪些能力

建议第一版范围收紧，否则会很快失控。

### 必做

- 单教材接入
- 单元列表展示
- 单元详情展示
- 基于结构化字段生成 system prompt
- 文本对话式口语对练
- 结束后的覆盖情况回显

### 暂缓

- 多教材后台批量导入
- 复杂评分体系
- 语音识别
- 语音合成
- 自动纠错报告细分到语法类型
- 家长端或老师端数据统计

## 8. 当前内容设计原则

后续补全和修改时，建议遵守这些规则：

### 8.1 参考内容不是硬模板

允许：

- 调整任务描述顺序
- 补全缺失词汇或例句
- 改写更自然的 prompt 步骤
- 调整展示方式

不建议：

- 改掉教材核心知识点
- 删除单元主任务
- 把低龄学生对话写成长句

### 8.2 prompt 配置优先结构化，不优先长文本

优先存：

- 数组
- 枚举值
- 独立字段

尽量不要只存一大段难拆分的 prompt。

### 8.3 页面展示优先“学习理解”，不是“内容堆砌”

例如单元详情页不应该原样把全部 prompt 倒给用户，而应该转换成：

- 本节练什么
- 你会用到哪些句子
- 你会遇到哪些词
- 你会怎么开始说

## 9. 当前最值得先确定的 4 个问题

这 4 个问题会直接决定后续实现方式：

1. 第一版是不是只先做 `人教版二年级上册`
2. 第一版是不是只做 `文字聊天`，暂不接语音
3. 第一版用户是不是默认 `学生`，暂不区分老师端和家长端
4. 复习单元是否第一版一起做，还是先只做普通单元
