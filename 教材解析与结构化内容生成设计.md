# 教材解析与结构化内容生成设计

## 1. 产品定位

当前系统应定义为一个 `通过导入英文教材 PDF，自动解析单元内容，并生成结构化教学板块的 Web 工具`。

产品定义中不包含：

- 对练聊天页
- 练习过程管理
- 学生端运行时

当前需要聚焦的是：

1. 上传 PDF 英文教材
2. 自动识别教材名称、单元编号、单元名称
3. 自动抽取或生成单词、重点句型、例句、对话样例、单元任务介绍、提示等板块
4. 对生成结果进行审核
5. 结构化展示结果
6. 支持下载 JSON 或 Markdown

## 2. 核心原则

### 2.1 解析优先，生成其次

每个板块的前提都必须是 `以解析内容为依据`。

正确链路应该是：

1. 先从 PDF 里解析到教材原始内容
2. 把原始内容按单元和板块做结构化归类
3. 再基于这些已解析内容，使用对应 prompt 做规范化、补全、摘要或重组
4. 对生成结果进行人工审核
5. 最后输出统一格式的结构化结果

这意味着 prompt 的作用不是“凭空写内容”，而是：

- 对解析结果做归纳和重写
- 补齐更适合展示的结构
- 把原始教材内容转成标准字段
- 在受控范围内生成“单元任务介绍”“提示”等衍生板块

### 2.2 生成内容必须可追溯

建议每个输出板块都保留：

- `source_pages`
- `source_excerpt`
- `confidence`
- `generation_mode`

其中：

- `generation_mode=extracted` 表示直接抽取
- `generation_mode=normalized` 表示基于原文规范化整理
- `generation_mode=derived` 表示基于解析内容推导生成

### 2.3 prompt 是“板块控制器”，不是“大一统长 prompt”

不要只写一个总 prompt 一次性生成所有内容。

建议按板块拆分 prompt：

- 教材识别 prompt
- 单元识别 prompt
- 词汇整理 prompt
- 句型整理 prompt
- 对话样例整理 prompt
- 单元任务介绍生成 prompt
- 单元提示生成 prompt

## 3. 目标输出板块

系统最终至少输出这些结构化板块。

### 3.1 教材信息

- 教材名称
- 教材版本

### 3.2 单元信息

- 单元编号
- 单元名称
- 单元主题

说明：

- 教材版本、教材名称、单元编号、单元名称是每个导出板块必须具备的分类信息

### 3.3 单词板块 `vocabulary`

每个词条建议字段：

- `classification`
- `word`
- `part_of_speech`
- `meaning_zh`
- `example_sentences`
- `source_pages`
- `confidence`

### 3.4 句型板块 `sentence_patterns`

每个句型建议字段：

- `classification`
- `pattern`
- `usage_note`
- `examples`
- `source_pages`
- `confidence`

说明：

- 每个重点句型建议配 `1-2` 个例句

### 3.5 对话样例板块 `dialogue_samples`

每个样例建议字段：

- `classification`
- `turns`
- `source_pages`
- `confidence`

说明：

- 每个对话样例建议为 `10-15` 轮
- 每一轮都必须有英文和中文
- 对话内容必须依据单元词汇、句型和主题生成或整理

### 3.6 单元任务介绍板块 `unit_task`

建议字段：

- `classification`
- `task_intro`
- `source_basis`
- `confidence`

说明：

- 这一部分通常不是逐字抽取
- `task_intro` 建议控制在 `20` 字左右
- 更适合根据词汇、句型、对话样例和单元标题做受控生成

### 3.7 提示板块 `unit_prompt`

建议字段：

- `classification`
- `unit_theme`
- `grammar_rules`
- `prompt_notes`
- `source_basis`
- `confidence`

说明：

- 这一部分用于承载单元主题和语法规则抽取
- 应该由 prompt 根据解析后的内容做推导生成
- 不能脱离当前单元内容凭空扩写

## 4. 整体系统链路

建议把系统拆成 6 段流程：

1. 文件接入
2. PDF 解析
3. 内容结构化
4. 板块生成
5. 结果审核
6. 结果输出

### 4.1 文件接入

输入：

- PDF 文件

处理：

- 文件上传
- 文件存储
- 创建解析任务

输出：

- `parse_job`

### 4.2 PDF 解析

建议拆分模块：

- `pdf_loader`
- `ocr_processor`
- `layout_analyzer`
- `unit_detector`
- `section_classifier`
- `vocabulary_extractor`
- `sentence_extractor`
- `dialogue_extractor`

这一步的目标不是直接生成最终成品，而是先拿到 `原始证据层`。

建议原始证据层包含：

- 页面文本块
- 标题层级
- 版面区域
- 疑似单元标题
- 疑似词汇区
- 疑似句型区
- 疑似对话区

### 4.3 内容结构化

这一步要把解析出来的证据组织成统一数据草稿。

建议形成：

- `book_draft`
- `unit_drafts`
- `vocabulary_drafts`
- `sentence_pattern_drafts`
- `dialogue_drafts`

此时的重点是：

- 按单元归属
- 去重
- 字段归一化
- 保留来源页

### 4.4 板块生成

这一步才引入 prompt。

每个板块各自处理：

- `book_normalizer`
- `unit_normalizer`
- `vocabulary_normalizer`
- `sentence_pattern_normalizer`
- `dialogue_normalizer`
- `task_generator`
- `prompt_generator`

其中：

- `normalizer` 负责规范化和轻量补全
- `generator` 负责生成任务介绍和提示等衍生内容

### 4.5 结果审核

这一步用于确认“生成结果是否可被导出或发布”。

建议审核能力至少包括：

- 查看原文依据
- 查看来源页码
- 查看生成模式
- 标记通过或驳回
- 填写审核备注
- 必要时人工修订字段

建议审核粒度至少支持：

- 按单元审核
- 按板块审核

### 4.6 结果输出

输出形态建议支持：

- 页面结构化预览
- JSON 下载
- Markdown 下载

约束：

- 仅允许审核通过的内容进入导出

## 6. prompt 控制策略

### 6.1 词汇板块 prompt

输入：

- 当前单元标题
- 当前单元词汇原始片段
- 页码信息

输出目标：

- 拆成标准词汇字段
- 统一词性格式
- 分离英音和美音
- 清洗例句

约束：

- 不新增教材中不存在的核心词汇
- 若音标缺失，可留空，不强造

### 6.2 句型板块 prompt

输入：

- 当前单元句型原始文本
- 所属单元标题
- 邻近例句

输出目标：

- 识别关键句型
- 抽取或重组成规范示例
- 必要时增加简短用法说明

约束：

- 不脱离单元主题扩写
- 优先保留教材原句

### 6.3 对话样例板块 prompt

输入：

- 对话原文块
- 说话人标记
- 页码信息

输出目标：

- 分离标题、角色和正文
- 中英文对齐
- 清理 OCR 噪声

约束：

- 不改写对话核心语义
- 不随意补新情节

### 6.4 单元任务介绍生成 prompt

输入：

- 单元标题
- 词汇板块
- 句型板块
- 对话样例板块

输出目标：

- 生成单元任务介绍

约束：

- 必须围绕当前单元可见内容
- 不能写出教材未覆盖的知识点

### 6.5 单元提示生成 prompt

输入：

- 单元标题
- 重点词汇
- 重点句型
- 对话样例

输出目标：

- 提炼单元主题
- 提炼语法规则
- 提炼补充提示

约束：

- 提示必须由当前单元内容推导
- 不要泛化成整学段通用空话

## 7. 页面信息架构

既然产品本身不包含互动运行时，页面应收敛为 5 个核心页面。

### 7.1 首页 / 上传页

目标：

- 上传 PDF
- 查看能力说明

核心模块：

- 文件上传
- 文件格式说明
- 支持抽取的板块说明
- 历史任务入口

### 7.2 解析任务页

目标：

- 查看处理进度

核心模块：

- 任务状态
- 进度条
- 当前阶段
- 错误信息
- 查看结果按钮

### 7.3 解析结果总览页

目标：

- 查看整本教材的结构化产出

核心模块：

- 教材信息
- 单元列表
- 各板块统计
- 审核状态汇总
- 导出状态提示

### 7.4 单元详情页

目标：

- 查看某个单元的完整结构化板块和来源依据

核心模块：

- 单元名称
- 单词板块
- 句型板块
- 对话样例板块
- 单元任务介绍板块
- 提示板块
- 来源页和置信度信息

### 7.5 审核与导出页

目标：

- 对结构化内容做人工确认后再导出

核心模块：

- 审核状态筛选
- 驳回和通过操作
- 审核备注
- 修订后预览
- JSON 下载
- Markdown 下载

## 8. MVP 范围

### 8.1 v1 必做

- 文本型 PDF 上传
- 教材名称识别
- 单元识别
- 词汇抽取
- 句型抽取
- 对话样例抽取
- 单元任务介绍生成
- 提示生成
- 结构化结果展示
- 基础审核状态流转
- 审核通过后导出控制
- JSON/Markdown 导出

### 8.2 v2 再做

- OCR 支持
- 扫描件适配
- 更强 layout 识别
- 多教材格式泛化

## 9. 当前最关键的实现约束

1. 抽取层和生成层必须分开，不要混写在一个步骤里
2. 结构化字段必须保留来源页和置信度
3. 单元任务介绍、提示属于 `derived content`，不能伪装成直接抽取结果
4. 审核必须发生在导出之前
5. 下载格式要和页面展示共用同一份标准数据

## 10. 下一步建议

建议下一步直接做这 3 件事：

1. 把上述对象细化成 JSON Schema
2. 设计 `upload / parse / result / download` 接口
3. 设计解析模块与生成模块的目录结构
