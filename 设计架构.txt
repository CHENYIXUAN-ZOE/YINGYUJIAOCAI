请你以harness工程思维帮我设计并实现一个“可外部访问的 Web 工具”，用于解析英文教材 PDF，并输出结构化内容。


重要：先做系统设计，不要直接写代码。最后再生成项目骨架。

==================================================
一、产品目标（必须理解）
==================================================

我要做一个在线工具，用户可以：

1. 通过浏览器访问一个网页（外部链接）
2. 上传 PDF 英文教材
3. 系统自动解析教材结构
4. 返回结构化结果（JSON / Markdown / 可下载文件）

这是一个完整系统，而不是脚本，需要包括：

- 前端（上传 + 展示结果）
- 后端 API
- PDF 解析服务
- 可部署（本地 / 云端）

==================================================
二、核心功能
==================================================

用户流程：

1. 打开网页
2. 上传 PDF
3. 点击“开始解析”
4. 后端处理
5. 页面展示解析结果
6. 支持下载 JSON / Markdown

==================================================
三、PDF 解析目标
==================================================

系统需要自动识别并结构化：

1. 教材名称
2. 单元（Unit / Lesson / Chapter）
   - 单元编号
   - 单元名称
3. 单词（vocabulary）
   - word
   - pos
   - phonetic
   - meaning
   - example
4. 句型（sentence patterns）
   - pattern
   - explanation
   - examples
5. 对话（dialogue）
   - title
   - speaker
   - text
6. 其他版块（可扩展）

要求：
- 自动识别结构（不是纯文本提取）
- 适配不同教材格式
- 支持扫描 PDF（OCR）

==================================================
四、系统架构要求
==================================================

请设计一个标准 Web 系统架构，包括：

1. 前端
   - 文件上传
   - 进度显示
   - 结果展示
   - 下载按钮

2. 后端 API
   - 上传接口
   - 解析接口
   - 查询结果接口

3. 解析服务（核心）
   - PDF 解析 pipeline
   - OCR
   - layout 分析
   - 单元识别
   - section 分类
   - 内容抽取

4. 存储（可选）
   - 本地文件
   - 或简单数据库（SQLite）
