## 一、项目当前定义

当前工具定义为：

`通过导入英文教材 PDF，自动解析教材内容，生成结构化教学板块，并在审核后导出的 Web 工具`

当前主链路固定为：

1. 上传 PDF
2. 解析教材内容
3. 按单元归类
4. 生成结构化板块
5. 人工审核
6. 导出 JSON / Markdown

当前不包含：

- 对练聊天页
- 学生端互动运行时
- 练习过程管理

## 二、当前确定的输出板块

每个导出板块都必须带同一组分类信息：

- `textbook_version`
- `textbook_name`
- `unit_code`
- `unit_name`

这组字段统一收在 `classification` 里。

### 1. 词汇 `vocabulary`

每条词汇至少包含：

- `word`
- `part_of_speech`
- `meaning_zh`
- `example_sentences`

规则：

- 例句建议 `1-2` 句
- 优先使用教材原句
- 若教材没有完整句，可基于单元内容补成简单句

### 2. 重点句型 `sentence_patterns`

每条句型至少包含：

- `pattern`
- `usage_note`
- `examples`

规则：

- 每单元建议输出 `3-5` 个重点句型
- 每个重点句型配 `1-2` 个例句
- 句型优先保留教材显式重复练习的核心表达

### 3. 对话样例 `dialogue_samples`

每条对话样例至少包含：

- `title`
- `turns`

每个 `turn` 必须包含：

- `turn_index`
- `speaker`
- `text_en`
- `text_zh`

规则：

- 每个对话样例必须是 `10-15` 轮
- 每一轮必须同时有英文和中文
- 对话内容必须依据单元词汇、句型和主题生成或整理
- v1 默认每单元输出 `1` 个主对话样例

### 4. 单元任务介绍 `unit_task`

至少包含：

- `task_intro`
- `source_basis`

规则：

- `task_intro` 以 `20` 字左右为目标
- 用一句短话说明“这个单元主要让学生做什么”
- 不能写成空泛目标

### 5. 提示 `unit_prompt`

至少包含：

- `unit_theme`
- `grammar_rules`
- `prompt_notes`
- `source_basis`

规则：

- `unit_theme` 用来抽取单元主题
- `grammar_rules` 用来抽取本单元语法规则
- `prompt_notes` 用来放补充提示
- 语法规则建议控制在 `1-3` 条

## 三、当前系统规则

### 1. 先解析，再生成

所有板块都必须先有教材解析依据，再做规范化或受控生成。

禁止：

- 跳过解析，直接凭 prompt 写内容
- 在没有依据的情况下扩写核心知识点

### 2. 生成模式

当前统一四类：

- `extracted`
- `normalized`
- `derived`
- `manual`

含义：

- `extracted`：直接抽取
- `normalized`：按原文整理
- `derived`：基于解析内容推导生成
- `manual`：人工修订

### 3. 审核后才能导出

当前规则是：

- `pending` 不导出
- `rejected` 不导出
- `approved` 可导出
- `revised` 是否导出后续再细定

## 四、当前数据结构

顶层数据结构固定为：

```json
{
  "job": {},
  "book": {},
  "units": [],
  "review_records": [],
  "export_meta": {}
}
```

其中：

- `job`：解析任务信息
- `book`：教材信息
- `units`：单元内容包
- `review_records`：审核流水
- `export_meta`：导出信息

### 单元内容包结构

每个 `unit` 的结构固定为：

```json
{
  "unit": {},
  "vocabulary": [],
  "sentence_patterns": [],
  "dialogue_samples": [],
  "unit_task": {},
  "unit_prompt": {}
}
```

## 五、当前审核重点

审核建议按这个顺序看：

1. 分类信息是否正确
2. 词汇是否正确
3. 重点句型是否正确
4. 对话样例是否覆盖主题
5. 单元任务介绍是否简洁准确
6. 提示是否提炼到位

## 六、板块级 Prompt 模板

下面这部分是下一步工作产物，作为生成层的直接模板基线。

### 1. 通用生成原则

所有板块的 prompt 都应遵守：

- 输入必须包含当前单元的解析依据
- 输出必须严格对应目标字段
- 不允许无依据扩写教材核心知识点
- 无法确认的信息允许留空，不要编造
- 生成内容必须与当前 `classification` 一致

### 2. 词汇板块 Prompt 模板

用途：

- 把当前单元词汇原始片段整理成标准词汇项

输入建议：

- `classification`
- 当前单元词汇原始文本
- 邻近页文本
- 页码信息

输出目标：

- `word`
- `part_of_speech`
- `meaning_zh`
- `example_sentences`

硬约束：

- 不新增教材中不存在的核心词汇
- `example_sentences` 建议输出 `1-2` 句
- 优先保留教材原句
- 词性统一成短格式

模板：

```text
你现在要基于教材解析结果，整理当前单元的词汇板块。

任务要求：
1. 只处理当前提供的单元内容。
2. 输出的每个词汇项必须属于当前 classification。
3. 每个词汇项包含：word, part_of_speech, meaning_zh, example_sentences。
4. example_sentences 输出 1-2 句，优先使用教材原句。
5. 如果词性或释义无法确定，可以留空，不要编造。
6. 不要新增教材中没有依据的核心词汇。

请只输出结构化结果，不要输出解释性文字。
```

### 3. 重点句型板块 Prompt 模板

用途：

- 从当前单元解析内容中筛出核心句型，并配 1-2 个例句

输入建议：

- `classification`
- 当前单元句型区原文
- 当前单元词汇摘要
- 当前单元对话摘要

输出目标：

- `pattern`
- `usage_note`
- `examples`

硬约束：

- 每单元优先输出 `3-5` 个重点句型
- 每条句型配 `1-2` 个例句
- 只保留本单元教学核心句型

模板：

```text
你现在要基于教材解析结果，整理当前单元的重点句型板块。

任务要求：
1. 从当前单元中筛选最核心的重点句型，不要罗列所有句子。
2. 优先输出 3-5 个重点句型；如果教材显式句型不足，则按实际数量输出。
3. 每个句型项包含：pattern, usage_note, examples。
4. examples 必须输出 1-2 句。
5. 例句必须和句型一致，并尽量贴合当前单元主题。
6. 不要把普通陈述句误判成重点句型。

请只输出结构化结果，不要输出解释性文字。
```

### 4. 对话样例板块 Prompt 模板

用途：

- 基于单元词汇、句型和原始对话依据，生成或整理一个示范对话

输入建议：

- `classification`
- 当前单元重点词汇
- 当前单元重点句型
- 当前单元对话原文或场景原文

输出目标：

- `title`
- `turns`

每个 `turn` 输出：

- `turn_index`
- `speaker`
- `text_en`
- `text_zh`

硬约束：

- 必须输出 `10-15` 轮
- 每轮都必须有中英文
- 内容必须围绕当前单元主题
- 尽量覆盖当前单元核心词汇和句型

模板：

```text
你现在要基于教材解析结果，整理或生成当前单元的对话样例板块。

任务要求：
1. 只围绕当前 classification 对应的单元内容生成。
2. 输出 1 个主对话样例。
3. 对话必须包含 10-15 轮。
4. 每轮都必须包含：turn_index, speaker, text_en, text_zh。
5. 对话要尽量覆盖当前单元的核心词汇和重点句型。
6. text_zh 只写对应句子的直接中文释义，不要额外扩写。
7. 如果教材已有对话，优先在其基础上整理；如果轮次不足，可基于单元内容补足。

请只输出结构化结果，不要输出解释性文字。
```

### 5. 单元任务介绍 Prompt 模板

用途：

- 生成一句短的单元任务介绍

输入建议：

- `classification`
- 单元标题
- 词汇摘要
- 句型摘要
- 对话样例摘要

输出目标：

- `task_intro`
- `source_basis`

硬约束：

- 目标长度约 `20` 字
- 必须说明学生在本单元主要做什么
- 不要写成空泛教学口号

模板：

```text
你现在要基于教材解析结果，生成当前单元的任务介绍。

任务要求：
1. 只根据当前单元已解析出的词汇、句型、对话和标题生成。
2. 输出字段为：task_intro, source_basis。
3. task_intro 用一句短话概括本单元主要表达任务，长度尽量控制在 20 字左右。
4. 不要写空泛目标，如“提高英语能力”“掌握英语知识”。
5. source_basis 要简要说明这条任务介绍依据了哪些单元内容。

请只输出结构化结果，不要输出解释性文字。
```

### 6. 提示板块 Prompt 模板

用途：

- 生成单元主题、语法规则和补充提示

输入建议：

- `classification`
- 单元标题
- 重点句型
- 重点词汇
- 对话样例

输出目标：

- `unit_theme`
- `grammar_rules`
- `prompt_notes`
- `source_basis`

硬约束：

- `unit_theme` 应是一个短主题
- `grammar_rules` 建议 `1-3` 条
- 只抽取当前单元真正相关的语法规则

模板：

```text
你现在要基于教材解析结果，生成当前单元的提示板块。

任务要求：
1. 输出字段为：unit_theme, grammar_rules, prompt_notes, source_basis。
2. unit_theme 用简短词组或短句概括当前单元主题。
3. grammar_rules 输出 1-3 条，只保留当前单元最相关的语法规则。
4. prompt_notes 可输出 0-3 条补充提示。
5. 不要输出整册级别的大语法总结，不要超出当前单元范围。
6. source_basis 要说明这些提示依据了哪些已解析内容。

请只输出结构化结果，不要输出解释性文字。
```

## 九、v1 默认数量约束与强制项

这一节先给出一版 `v1 默认值`，后续如果你要收紧或放宽，再在这里改。

### 1. 词汇 `vocabulary`

数量建议：

- 默认建议每单元输出 `8-15` 个词汇项
- 如果教材显式重点词不足，可少于 8 个
- 如果教材重点词明显较多，最多先控制在 `20` 个以内

强制项：

- 每个词汇项都必须有 `classification`
- 每个词汇项都必须有 `word`
- 每个词汇项都必须有 `meaning_zh`
- `example_sentences` 建议 `1-2` 句

### 2. 重点句型 `sentence_patterns`

数量建议：

- 默认建议每单元输出 `3-5` 个重点句型
- 如果教材显式句型不足，则按实际数量输出

强制项：

- 每个句型项都必须有 `classification`
- 每个句型项都必须有 `pattern`
- 每个句型项都必须有 `examples`
- `examples` 必须是 `1-2` 条

### 3. 对话样例 `dialogue_samples`

数量建议：

- v1 默认每单元输出 `1` 个主对话样例
- 如教材本身有两组非常清晰的对话，可扩展到 `2` 个

强制项：

- 每个对话样例都必须有 `classification`
- 每个对话样例都必须有 `turns`
- 每个对话样例必须为 `10-15` 轮
- 每一轮都必须有 `text_en` 和 `text_zh`

覆盖建议：

- 尽量覆盖至少 `2` 个重点句型
- 尽量覆盖至少 `4` 个核心词汇

### 4. 单元任务介绍 `unit_task`

数量建议：

- 每单元固定输出 `1` 条

强制项：

- 必须有 `classification`
- 必须有 `task_intro`
- 必须有 `source_basis`

长度建议：

- 目标 `20` 字左右
- 当前可接受范围：`15-30` 字

### 5. 提示 `unit_prompt`

数量建议：

- 每单元固定输出 `1` 组

强制项：

- 必须有 `classification`
- 必须有 `unit_theme`
- 必须有 `grammar_rules`
- 必须有 `source_basis`

数量建议：

- `grammar_rules` 建议 `1-3` 条
- `prompt_notes` 建议 `0-3` 条

## 十、审核判定规则表

### 1. 通用审核规则

| 审核项 | 通过标准 | 驳回标准 | 可人工修订项 |
| --- | --- | --- | --- |
| 分类信息 | 教材版本、教材名称、单元编号、单元名称都正确 | 分类归属错误或缺失 | classification |
| 来源信息 | 有来源页或来源依据 | 无法回溯来源 | source_pages, source_excerpt, source_basis |
| 生成方式 | generation_mode 与实际过程一致 | 把推导内容伪装成抽取内容 | generation_mode |
| 内容相关性 | 与当前单元强相关 | 明显偏离当前单元 | 各板块正文 |

### 2. 板块审核规则

| 板块 | 通过条件 | 驳回条件 | 备注 |
| --- | --- | --- | --- |
| vocabulary | 词汇属于本单元，词性和释义准确，例句可读 | 词汇偏题、词性错误、释义过泛、例句无关 | 优先保留教材重点词 |
| sentence_patterns | 句型是教学核心，例句 1-2 条且匹配句型 | 普通句误判成句型，例句缺失或超量 | 默认控制在 3-5 条 |
| dialogue_samples | 10-15 轮，中英文齐全，覆盖核心内容 | 轮次不足、缺中文、内容跑题 | v1 默认 1 个主样例 |
| unit_task | 一句话可读懂，能概括单元主要表达任务 | 太空、太长、看不出任务 | 目标约 20 字 |
| unit_prompt | 单元主题明确，语法规则适中且相关 | 规则太杂、太泛、超纲 | grammar_rules 建议 1-3 条 |

### 3. 审核状态流转建议

建议流程：

1. 初始生成后标记为 `pending`
2. 审核通过后标记为 `approved`
3. 审核驳回后标记为 `rejected`
4. 人工修改后标记为 `revised`
5. 如果 `revised` 内容再次确认通过，可转回 `approved`

## 十一、REST API 设计

接口建议统一前缀：

`/api/v1`

### 1. 上传 PDF

`POST /api/v1/upload`

用途：

- 上传教材 PDF
- 创建解析任务

请求：

- `multipart/form-data`
- 字段：`file`

响应建议：

```json
{
  "job_id": "job_001",
  "file_name": "grade2_book.pdf",
  "status": "uploaded",
  "progress": 0
}
```

### 2. 开始解析

`POST /api/v1/parse/{job_id}`

用途：

- 启动该任务的解析、结构化和生成流程

响应建议：

```json
{
  "job_id": "job_001",
  "status": "parsing"
}
```

### 3. 查询任务状态

`GET /api/v1/jobs/{job_id}`

用途：

- 轮询任务进度
- 查看是否已进入审核阶段

响应建议：

```json
{
  "job_id": "job_001",
  "status": "reviewing",
  "progress": 90,
  "review_status": "pending",
  "error_message": null
}
```

### 4. 获取整本结果

`GET /api/v1/results/{job_id}`

用途：

- 获取该任务的完整结构化结果

返回对象：

- 顶层结构应直接对齐 schema

```json
{
  "job": {},
  "book": {},
  "units": [],
  "review_records": [],
  "export_meta": {}
}
```

### 5. 获取单元结果

`GET /api/v1/results/{job_id}/units/{unit_id}`

用途：

- 单独查看某个单元内容包
- 支持单元级审核页

返回对象：

- 单个 `UnitPackage`

### 6. 更新单个内容项审核状态

`PATCH /api/v1/review/items/{target_type}/{target_id}`

用途：

- 审核单个板块或单个内容项

请求建议：

```json
{
  "review_status": "approved",
  "review_notes": "内容准确，可导出",
  "reviewer": "admin"
}
```

可用的 `target_type`：

- `book`
- `unit`
- `vocabulary_item`
- `sentence_pattern`
- `dialogue_sample`
- `unit_task`
- `unit_prompt`

### 7. 批量审核单元

`POST /api/v1/review/units/{unit_id}/batch`

用途：

- 一次性提交单元下多个板块的审核结果

适用场景：

- 审核页中统一通过
- 审核页中统一驳回

### 8. 导出结果

`POST /api/v1/export`

用途：

- 触发 JSON 或 Markdown 导出

请求建议：

```json
{
  "job_id": "job_001",
  "export_scope": "book",
  "approved_only": true,
  "format": "json"
}
```

约束：

- 默认只允许导出审核通过内容
- 若存在未审核或驳回内容，应返回拦截信息

响应建议：

```json
{
  "export_id": "exp_001",
  "status": "completed",
  "download_url": "/api/v1/export/exp_001/download"
}
```

### 9. 下载导出文件

`GET /api/v1/export/{export_id}/download`

用途：

## 十三、接口请求参数与错误码规范

这一节用于把上一节的 REST API 进一步定成可实现规范。

### 1. 通用响应结构

成功响应建议统一为：

```json
{
  "success": true,
  "data": {},
  "message": "ok"
}
```

失败响应建议统一为：

```json
{
  "success": false,
  "error": {
    "code": "JOB_NOT_FOUND",
    "message": "job_id does not exist",
    "details": {}
  }
}
```

### 2. 通用错误码

| 错误码 | 含义 | 推荐 HTTP 状态码 |
| --- | --- | --- |
| `INVALID_REQUEST` | 请求参数不合法 | `400` |
| `FILE_REQUIRED` | 未上传文件 | `400` |
| `UNSUPPORTED_FILE_TYPE` | 文件类型不支持 | `400` |
| `FILE_TOO_LARGE` | 文件过大 | `400` |
| `JOB_NOT_FOUND` | 任务不存在 | `404` |
| `UNIT_NOT_FOUND` | 单元不存在 | `404` |
| `TARGET_NOT_FOUND` | 审核目标不存在 | `404` |
| `PARSE_NOT_READY` | 结果尚未生成完成 | `409` |
| `REVIEW_BLOCKED` | 当前状态不允许审核 | `409` |
| `EXPORT_BLOCKED` | 存在未审核或驳回内容，不能导出 | `409` |
| `INVALID_REVIEW_STATUS` | 审核状态非法 | `400` |
| `INTERNAL_ERROR` | 服务内部异常 | `500` |

### 3. 上传接口参数规范

`POST /api/v1/upload`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `file` | file | 是 | 教材 PDF 文件 |

校验规则：

- 只允许 `.pdf`
- 文件不能为空
- 需要限制最大体积，v1 可先定为 `50MB`

失败场景：

- 缺文件 -> `FILE_REQUIRED`
- 非 PDF -> `UNSUPPORTED_FILE_TYPE`
- 超过大小 -> `FILE_TOO_LARGE`

### 4. 启动解析接口参数规范

`POST /api/v1/parse/{job_id}`

路径参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `job_id` | string | 是 | 上传后返回的任务 ID |

请求体建议：

```json
{
  "force_reparse": false
}
```

规则：

- `force_reparse=true` 表示允许重跑
- 如果任务不存在 -> `JOB_NOT_FOUND`
- 如果任务已在运行中且未允许重跑 -> `INVALID_REQUEST`

### 5. 查询任务状态接口参数规范

`GET /api/v1/jobs/{job_id}`

路径参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `job_id` | string | 是 | 任务 ID |

返回建议字段：

- `job_id`
- `status`
- `progress`
- `review_status`
- `error_message`
- `created_at`
- `finished_at`

### 6. 获取整本结果接口参数规范

`GET /api/v1/results/{job_id}`

路径参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `job_id` | string | 是 | 任务 ID |

查询参数建议：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `approved_only` | boolean | 否 | 是否只返回审核通过内容 |
| `include_review_records` | boolean | 否 | 是否返回审核流水 |

规则：

- 若任务未完成生成 -> `PARSE_NOT_READY`
- 默认 `approved_only=false`

### 7. 获取单元结果接口参数规范

`GET /api/v1/results/{job_id}/units/{unit_id}`

路径参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `job_id` | string | 是 | 任务 ID |
| `unit_id` | string | 是 | 单元 ID |

查询参数建议：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `approved_only` | boolean | 否 | 是否只返回审核通过内容 |

失败场景：

- 任务不存在 -> `JOB_NOT_FOUND`
- 单元不存在 -> `UNIT_NOT_FOUND`

### 8. 单项审核接口参数规范

`PATCH /api/v1/review/items/{target_type}/{target_id}`

路径参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `target_type` | string | 是 | 审核目标类型 |
| `target_id` | string | 是 | 审核目标 ID |

请求体建议：

```json
{
  "review_status": "approved",
  "review_notes": "内容准确，可导出",
  "reviewer": "admin",
  "patched_fields": {}
}
```

字段说明：

- `review_status`
  允许：`approved` `rejected` `revised`
- `review_notes`
  审核备注
- `reviewer`
  审核人标识
- `patched_fields`
  可选，用于人工修订字段

失败场景：

- 目标不存在 -> `TARGET_NOT_FOUND`
- 审核状态不合法 -> `INVALID_REVIEW_STATUS`

### 9. 单元批量审核接口参数规范

`POST /api/v1/review/units/{unit_id}/batch`

请求体建议：

```json
{
  "review_status": "approved",
  "targets": [
    {
      "target_type": "vocabulary_item",
      "target_id": "voc_001"
    },
    {
      "target_type": "sentence_pattern",
      "target_id": "sp_001"
    }
  ],
  "review_notes": "本单元整体通过",
  "reviewer": "admin"
}
```

适用场景：

- 批量通过
- 批量驳回
- 单元级审核收口

### 10. 导出接口参数规范

`POST /api/v1/export`

请求体建议：

```json
{
  "job_id": "job_001",
  "export_scope": "book",
  "approved_only": true,
  "format": "json",
  "unit_ids": []
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `job_id` | string | 是 | 任务 ID |
| `export_scope` | string | 是 | `book` 或 `unit` |
| `approved_only` | boolean | 是 | 是否只导出审核通过内容 |
| `format` | string | 是 | `json` 或 `markdown` |
| `unit_ids` | array | 否 | 单元级导出时指定单元列表 |

失败场景：

- 存在未审核内容 -> `EXPORT_BLOCKED`
- 导出范围非法 -> `INVALID_REQUEST`
