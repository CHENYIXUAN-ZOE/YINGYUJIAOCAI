对应基础设计：
[教材解析与结构化内容生成设计.md](/home/chenyixuan/口语对练/教材解析与结构化内容生成设计.md)

对应数据模型：
[教材结构化内容.schema.json](/home/chenyixuan/口语对练/教材结构化内容.schema.json)

## 1. 文档目的

这份文档用于进一步细化“生成和导出到底要长什么样”。

它解决的不是系统架构问题，而是板块级口径问题：

- 每个板块从哪里来
- 每个板块应该生成到什么程度
- 每个板块哪些内容允许推导
- 每个板块审核时看什么

当前先按 `v1 建议规则` 固化，后续可以继续收紧或放宽。

## 2. 总体规则

### 2.1 所有板块都必须携带分类信息

每个导出板块都必须能独立识别归属，因此必须携带：

- `textbook_version`
- `textbook_name`
- `unit_code`
- `unit_name`

这 4 个字段统一放在 `classification` 中。

### 2.2 先解析，再生成

统一链路：

1. 先从 PDF 解析出原始内容
2. 按单元归类
3. 再按板块做规范化或受控生成
4. 最后审核导出

禁止直接跳过解析、只靠 prompt 凭空产出内容。

### 2.3 生成优先级

各板块的生成优先级统一为：

1. `extracted`
2. `normalized`
3. `derived`
4. `manual`

解释：

- `extracted`
  直接从教材页面抽取
- `normalized`
  对教材原文做格式整理
- `derived`
  基于解析内容做受控推导
- `manual`
  人工修订

### 2.4 导出只出审核通过内容

统一规则：

- `pending` 不导出
- `rejected` 不导出
- `approved` 可导出
- `revised` 视业务规则决定是否先转 `approved`

### 2.5 原始依据必须可回溯

所有板块至少应保留：

- `source_pages`
- `source_excerpt`
- `confidence`
- `generation_mode`

如果一个板块是推导生成的，还应保留：

- `source_basis`

## 3. 板块一：词汇 `vocabulary`

### 3.1 板块目标

输出本单元最核心、最稳定、最适合教学展示的词汇信息。

### 3.2 来源优先级

优先从这些位置抽取：

1. 单词表或 vocabulary 区
2. 单元首页显式词汇区
3. 配图词汇标签
4. 重点句型和对话中高频出现、且明显属于教学目标的词

不建议从整页正文里盲目扩词。

### 3.3 字段要求

每条词汇输出：

- `word`
- `part_of_speech`
- `meaning_zh`
- `example_sentences`

附带控制字段：

- `classification`
- `source_pages`
- `source_excerpt`
- `confidence`
- `generation_mode`
- `review_status`

### 3.4 生成规则

- `word`
  保留教材标准拼写，不做同义替换
- `part_of_speech`
  统一成简洁标记，如 `n.` `v.` `adj.` `pron.` `prep.`
- `meaning_zh`
  用简洁中文释义，不写整句解释
- `example_sentences`
  建议 `1-2` 句

### 3.5 例句规则

词汇例句建议遵守：

- 优先使用教材原句
- 若教材只有短语没有完整句，可基于单元内容补成简单句
- 句长尽量短
- 尽量贴合单元主题
- 不要引入超出单元范围的新知识点

### 3.6 审核标准

通过标准：

- 词是本单元显式出现或高度明确属于该单元
- 词性正确
- 中文释义简洁准确
- 例句可读，且符合单元语境

驳回标准：

- 词汇明显不是本单元重点
- 词性错
- 释义过长或过泛
- 例句和单元内容无关

## 4. 板块二：重点句型 `sentence_patterns`

### 4.1 板块目标

输出本单元最值得教学使用的核心句型，而不是把所有句子都堆出来。

### 4.2 数量建议

v1 建议：

- 每单元优先输出 `3-5` 个重点句型
- 如果教材显式句型少于 3 个，则按教材原量输出

### 4.3 字段要求

每条句型输出：

- `pattern`
- `usage_note`
- `examples`

附带控制字段：

- `classification`
- `source_pages`
- `source_excerpt`
- `confidence`
- `generation_mode`
- `review_status`

### 4.4 生成规则

- `pattern`
  只保留句型骨架，不写成长段说明
- `usage_note`
  只写一句短说明，可为空
- `examples`
  必须是 `1-2` 个例句

### 4.5 句型筛选标准

优先保留：

- 教材显式重复练习的句型
- 单元标题和场景直接相关的句型
- 可以驱动整段对话生成的句型

不优先保留：

- 只出现一次的普通陈述句
- 与本单元核心目标无关的零散句子
- 过长、过复杂、明显超出年级水平的句子

### 4.6 审核标准

通过标准：

- 句型是本单元教学核心
- 例句和句型一致
- 例句数量为 `1-2`
- 表达简洁、适合展示

驳回标准：

- 把普通句子误当成重点句型
- 例句超过 2 条或没有例句
- 例句偏题

## 5. 板块三：对话样例 `dialogue_samples`

### 5.1 板块目标

输出一个能覆盖本单元主要词汇和句型的示范对话。

### 5.2 数量建议

v1 建议：

- 每单元默认输出 `1` 个主对话样例
- 若教材内本来就有多组清晰对话，可扩展为 `1-2` 个

### 5.3 轮次要求

每个对话样例必须：

- 为 `10-15` 轮
- 每轮都同时包含英文和中文
- 轮次连续编号

### 5.4 字段要求

每个对话样例输出：

- `title`
- `turns`

其中每个 `turn` 必须有：

- `turn_index`
- `speaker`
- `text_en`
- `text_zh`

附带控制字段：

- `classification`
- `source_pages`
- `source_excerpt`
- `confidence`
- `generation_mode`
- `review_status`

### 5.5 生成规则

- 优先使用教材现有对话做整理
- 如果教材原对话轮次不足，可基于单元词汇和句型补成完整教学对话
- 说话人默认 `A/B` 或沿用教材角色名
- 对话应至少覆盖多数重点句型中的核心表达
- 对话中应尽量复用本单元核心词汇

### 5.6 中文规则

`text_zh` 的要求：

- 对应当前句的直接中文释义
- 不做额外扩写
- 不把一句拆成多句解释

### 5.7 审核标准

通过标准：

- 轮次在 `10-15` 之间
- 每轮中英文齐全
- 对话和单元主题高度相关
- 至少覆盖若干核心词汇和句型

驳回标准：

- 轮次不足或过长
- 某些轮次没有中文
- 对话内容偏离单元
- 新增过多教材外内容

## 6. 板块四：单元任务介绍 `unit_task`

### 6.1 板块目标

用很短的一句话说明“这个单元主要让学生做什么”。

### 6.2 字数建议

v1 建议：

- 以 `20` 字左右为目标
- 实际可允许落在 `15-30` 字附近

### 6.3 字段要求

- `task_intro`
- `source_basis`

附带控制字段：

- `classification`
- `confidence`
- `generation_mode`
- `review_status`

### 6.4 生成规则

- 优先描述“学生要完成的表达任务”
- 尽量用动作性表述
- 不写成课程介绍长段落
- 不写空泛目标，如“掌握英语知识”

### 6.5 推荐写法

推荐结构：

- 围绕某主题进行表达
- 用某些句型完成简单交流
- 介绍某对象、某场景、某关系

示例风格：

- 围绕家庭成员进行简单介绍
- 用核心句型描述公园里的事物

### 6.6 审核标准

通过标准：

- 一句话可读懂
- 主题清晰
- 明确对应单元主要表达任务

驳回标准：

- 太空
- 太长
- 看不出和单元内容的关系

## 7. 板块五：提示 `unit_prompt`

### 7.1 板块目标

把本单元最适合展示给老师或审核者的“单元主题 + 语法规则 + 补充提示”抽出来。

### 7.2 字段要求

- `unit_theme`
- `grammar_rules`
- `prompt_notes`
- `source_basis`

附带控制字段：

- `classification`
- `confidence`
- `generation_mode`
- `review_status`

### 7.3 生成规则

- `unit_theme`
  应是一个短主题，不要写成长句
- `grammar_rules`
  建议 `1-3` 条
- `prompt_notes`
  建议 `0-3` 条

### 7.4 语法规则抽取原则

优先抽取：

- 教材显式强调的语法点
- 句型中反复出现的结构
- 对完成单元表达任务最关键的语法限制

不建议抽取：

- 整册级别的大语法总结
- 与当前单元关系弱的规则
- 需要跨很多单元才能理解的抽象规则

### 7.5 审核标准

通过标准：

- 单元主题简洁明确
- 语法规则数量适中
- 规则和当前单元高相关

驳回标准：

- 主题写成空话
- 语法规则过多或过杂
- 规则明显超纲

## 8. 推荐的板块生成顺序

建议系统内部按这个顺序生成：

1. `classification`
2. `vocabulary`
3. `sentence_patterns`
4. `dialogue_samples`
5. `unit_task`
6. `unit_prompt`

原因：

- 后两个板块明显依赖前面板块
- 对话样例也需要依赖词汇和句型

## 9. 审核顺序建议

建议审核页按这个顺序给审核人看：

1. 分类信息是否正确
2. 词汇是否正确
3. 重点句型是否正确
4. 对话样例是否覆盖主题
5. 单元任务介绍是否简洁准确
6. 提示是否提炼到位

## 10. 当前仍待继续对齐的点

这版已经先定了基础规则，但还有 5 个点值得继续细化：

1. 每单元词汇数量是否要限制上限
2. 重点句型到底固定 `3` 条、`5` 条，还是允许浮动
3. 对话样例默认输出 `1` 个还是 `2` 个
4. `task_intro` 是否要强控在更严格字数范围
5. `grammar_rules` 是否只允许抽取显式教材规则，还是允许轻度推导

## 11. 下一步建议

下一步最合适的是直接补一份：

- `板块级 prompt 模板设计`

把每个板块的输入、输出、硬约束和禁止项写成可直接给模型用的模板。
